# Tickets

## Purpose

This folder contains feature tickets and implementation specifications for the Basketball Analytics Platform.

| File | Description | Status |
|------|-------------|--------|
| [mention-autocomplete.md](./mention-autocomplete.md) | @-mention autocomplete with hierarchical browser | Planned |

## Ticket Format

Each ticket should include:
- **Overview**: What the feature does
- **User Experience**: How users interact with it
- **Technical Design**: Backend and frontend implementation details
- **File Changes**: What files need to be created/modified
- **Testing Requirements**: Unit, integration, and E2E tests
- **Implementation Order**: Phased approach
- **Acceptance Criteria**: Definition of done

## Workflow

1. Create ticket in this folder
2. Review and refine requirements
3. Create feature branch: `feature/<ticket-name>`
4. Implement following the phases outlined
5. Create PR referencing the ticket
6. Update ticket status after merge
